# 网站安全防护系统

## 概述

这是一个针对Flask应用的综合安全防护系统，主要用于防止路径扫描、恶意访问和其他常见的网络攻击。

## 功能特性

### 🛡️ 核心防护功能

1. **路径扫描防护**
   - 检测常见的扫描路径模式
   - 识别目录遍历攻击
   - 阻止对敏感文件的访问

2. **速率限制**
   - 限制单个IP的请求频率
   - 防止暴力破解和DDoS攻击
   - 可配置的时间窗口和请求限制

3. **恶意User-Agent检测**
   - 识别扫描工具的User-Agent
   - 阻止自动化攻击工具
   - 检测可疑的请求头

4. **IP封禁系统**
   - 自动封禁可疑IP
   - 支持临时和永久封禁
   - 白名单机制保护管理员IP

5. **蜜罐陷阱**
   - 设置诱饵路径捕获攻击者
   - 自动记录攻击行为
   - 立即封禁触发蜜罐的IP

### 📊 监控和管理

1. **实时监控面板**
   - 查看当前封禁IP列表
   - 监控可疑活动统计
   - 实时安全状态展示

2. **详细日志记录**
   - 记录所有安全事件
   - 攻击模式分析
   - 便于后续安全审计

## 安装和配置

### 1. 文件结构
```
stockdata/
├── security_middleware.py    # 核心安全中间件
├── security_config.py       # 安全配置文件
├── templates/
│   └── security_dashboard.html  # 安全监控面板
└── main.py                  # 主应用文件（已集成安全功能）
```

### 2. 配置说明

编辑 `security_config.py` 文件来调整安全参数：

```python
SECURITY_CONFIG = {
    'RATE_LIMIT_REQUESTS': 30,      # 每分钟最大请求数
    'RATE_LIMIT_WINDOW': 60,        # 时间窗口（秒）
    'SUSPICIOUS_THRESHOLD': 5,       # 可疑路径访问阈值
    'BAN_DURATION': 3600,           # 封禁时长（秒）
    'MAX_PATH_LENGTH': 200,         # 最大路径长度
}
```

### 3. 白名单配置

在 `security_config.py` 中添加你的管理IP到白名单：

```python
WHITELIST_IPS = [
    '127.0.0.1',
    '::1',
    '192.168.1.100',  # 添加你的管理IP
]
```

## 使用方法

### 1. 访问安全监控面板

以管理员身份登录后，访问：
```
https://your-domain.com/security/dashboard
```

### 2. 查看安全状态API

```
GET /security/status
```

### 3. 日志监控

安全事件会记录到应用日志中，包括：
- 封禁的IP地址
- 可疑路径访问
- 恶意User-Agent检测
- 蜜罐触发事件

## 防护机制详解

### 1. 路径扫描检测

系统会检测以下类型的可疑路径：
- PHP、ASP、JSP等脚本文件
- 管理面板路径（/admin, /wp-admin等）
- 配置文件（.env, .git等）
- 数据库管理工具路径
- 目录遍历尝试

### 2. 蜜罐路径

以下路径被设置为蜜罐，访问即触发封禁：
- `/admin`
- `/wp-admin`
- `/phpmyadmin`
- `/.env`
- `/config.php`
- 等等...

### 3. 封禁策略

- **临时封禁**：速率限制触发时封禁5分钟
- **永久封禁**：可疑行为达到阈值时封禁1小时
- **立即封禁**：触发蜜罐陷阱时立即封禁

## 安全建议

### 1. 定期维护
- 定期检查封禁IP列表
- 更新扫描模式和规则
- 监控日志中的新攻击模式

### 2. 配置优化
- 根据网站流量调整速率限制
- 添加业务相关的白名单路径
- 定期更新蜜罐路径

### 3. 额外防护
- 在服务器层面配置防火墙
- 使用CDN服务进行DDoS防护
- 定期更新系统和依赖包

## 故障排除

### 1. 误封问题
如果正常用户被误封，可以：
- 将用户IP添加到白名单
- 调整可疑行为阈值
- 检查扫描模式是否过于严格

### 2. 性能影响
如果安全检查影响性能：
- 优化正则表达式模式
- 增加白名单路径
- 调整检查频率

### 3. 日志过多
如果安全日志过多：
- 调整日志级别
- 设置日志轮转
- 过滤不重要的事件
