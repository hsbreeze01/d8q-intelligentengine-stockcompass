# Compass 策略组引擎 — 前端交互设计稿

> 文档版本: v2.0 | 日期: 2026-05-14
> 技术栈: Jinja2 + ECharts + fetch API + EventSource(SSE)
> 前置文档: [策略组产品需求](./Strategy_Group_Product_Requirements.md) | [全站交互设计](./d8q-interaction-design.md)
> 设计原则: 融入现有 Compass 体系，复用 dashboard.html 的视觉语言
> v2.0 变更: 角色分流（Admin 策略管理 vs User 发现/订阅/跟踪）+ 事件详情三维度（微观/宏观/信息关联）

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **融入现有体系** | 复用 dashboard.html 的色彩、卡片、网格布局，不引入新设计语言 |
| **数据驱动** | 所有页面以数据卡片 + ECharts 图表为主，减少文字描述 |
| **渐进式披露** | 列表 → 筛选 → 详情 → 操作，不一次性暴露所有信息 |
| **实时感** | SSE 推送信号更新，页面数据自动刷新，无需手动 F5 |
| **移动端适配** | CSS Grid auto-fit，核心功能手机可用 |

## 2. 角色模型与导航

### 2.1 角色定义

| 角色 | 权限 | 导航入口 |
|------|------|---------|
| **Admin / Editor** | 创建/编辑/删除/启停策略组，查看运行状态，管理生命周期 | 策略管理、事件跟踪 |
| **User（普通用户）** | 浏览可订阅策略，订阅/取消订阅，查看已订阅策略的事件，查看微观/宏观/信息关联 | 策略发现、我的策略 |

### 2.2 用户流程

```
Admin/Editor:
  策略管理 → 创建/编辑策略 → 配置指标+规则 → 启停 → 查看运行状态

User:
  策略发现 → 浏览策略 → 订阅 → 我的策略 → 查看策略事件 → 进入事件详情
                                                                ├─ 微观数据（个股指标快照）
                                                                ├─ 宏观数据（板块/行业趋势）
                                                                └─ 信息关联（资讯+LLM分析）
```

### 2.3 全局导航扩展

侧边栏增加角色切换（用户/管理），根据角色显示不同导航项：


## 6. 技术实现要点

### 6.1 Flask SSE 支持

```python
# compass/strategy/routes/signals.py
from flask import Response, stream_with_context
import json, time

@bp.route('/api/strategy/signals/stream')
def signal_stream():
    def generate():
        q = signal_queue  # 全局队列，scanner 写入
        while True:
            signal = q.get(timeout=30)
            yield f"data: {json.dumps(signal)}\n\n"
    return Response(
        stream_with_context(generate()),
        mimetype='text/event-stream',
        headers={'Cache-Control': 'no-cache', 'X-Accel-Buffering': 'no'}
    )
```

### 6.2 前端 EventSource

```javascript
const source = new EventSource('/api/strategy/signals/stream');
source.onmessage = (e) => {
    const signal = JSON.parse(e.data);
    prependSignalRow(signal);  // 插入表格顶部
    updateStatsCards();         // 更新指标卡
};
source.onerror = () => {
    showStatus('🔴 已断开，5s 后重连...');
    setTimeout(() => source.close(), 5000);
};
```
